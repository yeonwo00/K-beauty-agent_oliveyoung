# Demo

## English Request

```bash
curl -X POST http://127.0.0.1:8000/api/recommend \
  -H "Content-Type: application/json" \
  -d '{
    "skin_type": "oily",
    "concerns": ["oil_control", "pores"],
    "preferences": ["lightweight"],
    "language": "en"
  }'
```

Expected result:

- The API returns `Green Tea Oil-Control Gel Cream` or another oily-skin product near the top.
- `summary` explains why the first product is the strongest rule-based match.
- `why_recommended` explains why each product fits the entered conditions. With `OPENAI_API_KEY`, this is generated by the LLM from the rule evidence.
- `review` includes one positive review, one critical review, and an overall review summary.
- `rule_trace` shows the normalized skin type, concerns, preferences, and avoided ingredients.

## Korean Request

```bash
curl -X POST http://127.0.0.1:8000/api/recommend \
  -H "Content-Type: application/json" \
  -d '{
    "skin_type": "지성",
    "concerns": ["유분", "모공"],
    "preferences": ["산뜻"],
    "language": "ko"
  }'
```

Expected result:

- Korean aliases are normalized into rule-friendly tags such as `oily`, `oil_control`, and `pores`.
- The service returns a Korean summary when `language` is `ko`.
- The recommendation still exposes English product and ingredient names for portfolio readability.

## LLM Behavior

If `OPENAI_API_KEY` is set, the workflow asks the LLM to generate a short grounded summary from the rule-based matches.

If the key is missing or the API call fails, the system falls back to a deterministic rule-based summary.

## Product Comparison

```bash
curl -X POST http://127.0.0.1:8000/api/compare \
  -H "Content-Type: application/json" \
  -d '{
    "product_ids": ["kb-001", "kb-004"],
    "skin_type": "oily",
    "concerns": ["pores", "oil_control"],
    "preferences": ["lightweight"],
    "language": "en"
  }'
```

Expected result:

- The response contains the selected products and their ingredients, skin type fit, functions, price, and review snapshot.
- With `OPENAI_API_KEY`, the `comparison` text is generated by the LLM.
- Without the key, the API returns a deterministic comparison so the demo still works locally.
